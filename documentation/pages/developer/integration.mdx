# Integration

Integration with IOTA-Names works in multiple ways. For dApps, you typically integrate IOTA-Names through off-chain resolution using remote procedural calls (RPCs). The calls enable you to interact with the IOTA network or display names instead of addresses. For example, your RPCs might create transactions that send assets to names, or show names instead of addresses on a leaderboard.

Resolution works in two ways:

- Lookup: A name can point to an address or an object (target address).
- Reverse lookup: An address can have a default name.

## Addresses

Lookups work with two types of addresses:

- **Target address:** The address that an IOTA name resolves to. For example, `example.iota` might point to `0x2`, making `0x2` the target address for `example.iota`. Lookup resolution retrieves this information.
- **Default address:** The IOTA name that the owner of a particular address has selected to represent that address. For example, if you own `0x2` you can make `example.iota` its default address. The owner must sign and execute a "set default" transaction to establish this connection. The default address resets anytime the target address changes. Reverse lookup resolution retrieves this name.

        
## IOTA-Names NFT ownership

Don't use ownership of the IOTA-Names NFT as a resolution method. The NFT is used as the key (capability) to change the target address, but shouldn’t be used to identify any name with an address.

IOTA-Names NFT ownership allows any address to be set as the target address. So, the `example.iota` address used in the previous section can point to any address, not just `0x2`. Consequently, when you want to display default addresses, you should trust the default address over target address because it's guaranteed on chain. 

## Off-chain resolution

For off-chain resolution, use the available IOTA API endpoints: 

- [JSON-RPC](https://docs.iota.org/iota-api-ref#iotax_iotanameslookup)
- [GraphQL](https://docs.iota.org/developer/references/iota-api/iota-graphql/reference/devnet/operations/queries/resolve-iota-names-address)

<Callout type="info">
For GraphQL's default name resolution, there's a [`defaultIotaName` field](https://docs.iota.org/references/iota-api/iota-graphql/reference/types/objects/address) you can use.
</Callout>

## On-chain resolution

Use the `iota-names` **core package** as a dependency in your Move package for on-chain name resolution.

Testnet, Devnet, and Mainnet are supported. Add the following to your `Move.toml`:

```toml
[dependencies]
IotaNames = { git = "https://github.com/iotaledger/iota-names/", subdir = "packages/iota-names", rev = "develop" }
```


For more implementation details, review the [available code examples](./examples.mdx) that use the IOTA-Names core package for name resolution.

<Callout type="warning" emoji="⚠️">
  You should integrate IOTA-Names on chain using the core package only. The utility packages are likely to be replaced, rendering your logic non-functional unless you update your code with each change. 
</Callout>
