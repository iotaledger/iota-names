# IOTA-Names SDK

Use the IOTA-Names SDK to interact with the IOTA-Names. Query IOTA-Names data in a usable way and build transactions that interact with service names.

## Installation

To use with the latest version of the TypeScript SDK, install using:
```sh npm2yarn
npm i @iota/iota-names-sdk
```


## `IotaNamesClient`

`IotaNamesClient` is the base for all IOTA-Names functionality.

<Callout type="info" emoji="ℹ️">
    You should keep only one instance of `IotaNamesClient` throughout your dApp, API, or script. For example, in React, you should use a context to provide the client.
</Callout>

## Initializing an `IotaNamesClient`

<Callout type="warning" emoji="ℹ️">
    Always keep the dependency updated so you get the latest constants. If you don't, some of your transactions might fail to build.
</Callout>

You can initialize an `IotaNamesClient` by either providing the active network (`mainnet`/`testnet`),
or by passing in the constants (usable for any network).

<Tabs items={['Network initializer', 'Testnet Constants', 'Devnet Constants']}>
    <Tabs.Tab>
```js
import { IotaNamesClient } from '@iota/iota-names-sdk';
import { getFullnodeUrl, IotaClient } from '@iota/iota-sdk/client';
 
// You need an IOTA client. You can re-use the IOTA client of your project
// (it's not recommended to create a new one).
const client = new IotaClient({ url: getFullnodeUrl('testnet') });
 
// Now you can use it to create an IOTA-Names client.
const iotaNamesClient = new IotaNamesClient({
	client,
	network: 'testnet',
});
```
