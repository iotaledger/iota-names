# IOTANS SDK

Use the IOTANS SDK to interact with the IOTA Name Service. Query IOTANS data in a usable way and build transactions that interact with service names.

## Installation

<Tabs items={['Using @iota/iota-sdk', 'Using @iota/iota-sdk.js']}>
    <Tabs.Tab>
    To use with the latest version of the Typescript SDK, install using:
    ```sh npm2yarn
    npm i @iota/iota-ns
    ```
    </Tabs.Tab>
      <Tabs.Tab>
      To use with the older version (prior to 1.x) of the Typescript SDK (0.54.1), install using:
      ```sh npm2yarn
    npm i @iota/iota-ns@0.0.3
    ```
    </Tabs.Tab>

    </Tabs>



## `IotansClient`

`IotansClient` is the base for all IOTANS functionality.

<Callout type="info" emoji="ℹ️">
    You should keep only one instance of `IotansClient` throughout your dApp, API, or script. For example, in React, you should use a context to provide the client.
</Callout>

## Initializing a `IotansClient`

<Callout type="warning" emoji="ℹ️">
    Always keep the dependency updated so you get the latest constants. If you don't, some of your transactions might fail to build.
</Callout>

You can initialize a `IotansClient` by either providing the active network (`mainnet`/`testnet`),
or by passing in the constants (usable for any network).

<Tabs items={['Network initializer', 'Constants initializer']}>
    <Tabs.Tab>
```js
import { IotansClient } from '@iota/iota-ns';
import { getFullnodeUrl, IotaClient } from '@iota/iota-sdk/client';
 
// You need a IOTA client. You can re-use the IOTA client of your project
// (it's not recommended to create a new one).
const client = new IotaClient({ url: getFullnodeUrl('testnet') });
 
// Now you can use it to create a IOTANS client.
const iotansClient = new IotansClient({
	client,
	network: 'testnet',
});
```
